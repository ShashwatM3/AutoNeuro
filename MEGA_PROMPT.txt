You are being given an existing codebase skeleton called AutoNeuro. Your job is to convert it into a fully functional, plug-and-play agentic research pipeline. The pipeline must run a continuous optimization loop — calling Claude agents via the Anthropic Python SDK — with a working dashboard for human oversight. Below are your exact instructions.

0. Understand the architecture first
The system has these roles:

orchestrator.py — the main loop. Reads state, decides which agent to call, calls wrapper.sh, commits or reverts git changes.
wrapper.sh — runs train.py, writes to ERROR.txt and results.tsv, exits 0 on success.
agents/contact_agent.py — writes flags to SQLite, generates AI summaries. Already partially implemented.
dashboard/server.py — Flask server serving the dashboard. Already partially implemented.
state.json — ONLY stores: error_counter, iteration, current_best. Nothing else.
database.db — ONLY used by the dashboard. Stores runs and flags tables for the UI. Never read by the orchestrator directly.
results.tsv — the orchestrator's source of truth for metrics and status. Git-tracked. The orchestrator reads this, NOT the database.
ERROR.txt — written by wrapper.sh on crash. Read by orchestrator. Cleared by wrapper.sh at start of each run.
HUMAN_INSTRUCTION.txt — written by dashboard/server.py when a human responds to a flag. Read and cleared by orchestrator.

Critical rule: no overlap in purpose. state.json is for the loop's runtime counters only. database.db is for the dashboard UI only. results.tsv is the orchestrator's decision-making log. Do not mix these.

1. Fix the inconsistency: database.db vs state.json
The current codebase has both state.json and database.db. They have distinct, non-overlapping roles — keep both, but enforce clean boundaries:

state.json must only ever contain: { "error_counter": int, "iteration": int, "current_best": float | null }. The orchestrator reads and writes this file directly in Python. No other file reads or writes it.
database.db must only be written by agents/contact_agent.py (for flags) and by the orchestrator (after each run, to sync the runs table for the dashboard). The orchestrator must never make decisions based on database.db.
Create database.db with a proper schema initialisation function in agents/contact_agent.py (or a new db.py) that creates both tables if they don't exist:

2. Implement orchestrator.py fully
The orchestrator is the heartbeat of the system. Implement the following loop exactly:
on startup:
  init_db()
  load state.json → { error_counter, iteration, current_best }

loop forever:
  # Step 1: check for human instruction override
  if HUMAN_INSTRUCTION.txt is non-empty:
    human_instruction = read and clear HUMAN_INSTRUCTION.txt
  else:
    human_instruction = None

  # Step 2: read results.tsv
  rows = parse results.tsv (skip header)
  n = len(rows)

  # Step 3: decide what to call coding agent with
  if n == 0:
    call_coding_agent(mode="FIRST_RUN", human_instruction=human_instruction)
  elif n == 1:
    call_coding_agent(mode="SECOND_RUN", human_instruction=human_instruction)
  else:
    if ERROR.txt is non-empty:
      error_text = read ERROR.txt
      if state["error_counter"] >= 5:
        call_contact_agent(trigger="ERROR_LIMIT")
        wait_for_human_instruction()   # block polling HUMAN_INSTRUCTION.txt every 30s
      else:
        call_coding_agent(mode="FIX_ERROR", error=error_text, human_instruction=human_instruction)
    else:
      last = rows[-1]; prev = rows[-2]
      if float(last.metric) > float(prev.metric):   # higher is better — adjust sign if needed
        git_commit(f"iteration {state['iteration']}: KEEP")
        update results.tsv last row status → KEEP
        state["current_best"] = max(state["current_best"] or 0, float(last.metric))
      else:
        git_checkout_discard()   # git checkout -- train.py prepare.py
        update results.tsv last row status → DISCARD
      call_coding_agent(mode="OPTIMIZE", human_instruction=human_instruction)

  # Step 4: run wrapper.sh
  subprocess.run(["bash", "wrapper.sh"])

  # Step 5: sync run to database.db for dashboard
  sync_latest_run_to_db()

  # Step 6: update state.json
  if last run was CRASH:
    state["error_counter"] += 1
  else:
    state["error_counter"] = 0
  state["iteration"] += 1
  save state.json

3. Implement the Coding Agent call in orchestrator.py
Use the Anthropic Python SDK directly. Do not use any wrapper library. The pattern is:
pythonimport anthropic
import os

client = anthropic.Anthropic(api_key=os.environ["ANTHROPIC_API_KEY"])

def call_coding_agent(mode: str, error: str = None, human_instruction: str = None):
    system_prompt = open("agents/coding_agent.md").read()
    train_code = open("train.py").read()
    prepare_code = open("prepare.py").read()
    results = open("results.tsv").read()
    program = open("program.md").read()

    if mode == "FIRST_RUN":
        user_msg = f"""This is the very first iteration. Here is the current codebase.

program.md:
{program}

train.py:
{train_code}

prepare.py:
{prepare_code}

Do not change anything yet. Just confirm you have read the files and are ready."""

    elif mode == "SECOND_RUN":
        user_msg = f"""This is only the second iteration. We have one result but no comparison yet.
You are free to make one change you think will improve the metric.

program.md:
{program}

train.py:
{train_code}

prepare.py:
{prepare_code}

results.tsv:
{results}

Make exactly one change. Write the full updated content of whichever file(s) you modify."""

    elif mode == "FIX_ERROR":
        user_msg = f"""The last run crashed. Fix the error.

ERROR:
{error}

train.py:
{train_code}

prepare.py:
{prepare_code}

Write the full corrected file(s)."""

    elif mode == "OPTIMIZE":
        user_msg = f"""Propose and implement one optimization to improve the metric.

program.md:
{program}

train.py:
{train_code}

prepare.py:
{prepare_code}

results.tsv:
{results}

{"HUMAN INSTRUCTION (high priority): " + human_instruction if human_instruction else ""}

Write the full updated content of whichever file(s) you modify."""

    response = client.messages.create(
        model="claude-sonnet-4-20250514",
        max_tokens=8096,
        system=system_prompt,
        messages=[{"role": "user", "content": user_msg}]
    )

    raw = response.content[0].text

    # Parse and write back files
    # The coding agent must be instructed (in coding_agent.md) to wrap file contents like:
    # <file name="train.py">...</file>
    # Parse those tags and overwrite the files.
    import re
    for match in re.finditer(r'<file name="(train\.py|prepare\.py)">(.*?)</file>', raw, re.DOTALL):
        fname, content = match.group(1), match.group(2).strip()
        open(fname, "w").write(content)
        print(f"[orchestrator] wrote {fname}")
